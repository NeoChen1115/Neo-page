# 决策树预测问题分析报告

## 🔴 问题严重程度

### 0288.HK - 中等问题
```
实际价格范围:  [4.15, 8.56]
预测价格范围:  [4.04, 6.25] ❌ 无法覆盖上界
预测值唯一数量: 59个（原应364个）
最常见预测值:  5.92（出现34.6%）
平均误差:      -0.51元 (-8.6%)
```
**症状**: 决策树将多个样本映射到同一预测值

### 0005.HK - 严重问题 🔴
```
实际价格范围:  [57.78, 110.80]
预测价格范围:  [55.58, 55.58] ⚠️ 完全常数！
预测值唯一数量: 1个（原应358个）❌❌❌
常数预测值:    55.58（出现100%）
平均误差:      -24.01元 (-30.2%) 🔴严重
```
**症状**: 决策树完全退化成常数预测器 - 最严重的失败

---

## 🔍 根本原因分析

### 原因1: 【最可能】多特征输入导致的问题
```
当前特征: Close + High (对0005.HK)
问题:
- 多特征拆平后，特征矩阵变成: [30个Close, 30个High]
- 决策树难以学到这种复杂的特征交互
- 结果: 树完全学不到有用的模式
- 退化: 用平均值或某个叶子节点的值作为预测
```

### 原因2: 【次可能】决策树超参数不适合
```
当前配置:
- max_depth=10: 可能太深，导致过度拟合（在训练集上）
- min_samples_split=5: 太小，允许过度细分
- min_samples_leaf=1（默认）: 太小，可以创建单样本叶子

症状:
- 在训练集上学到很多"规则"
- 在测试集上，这些规则不适用 → 某些分支无样本通过
- 结果: 很多样本最后都打到同一个叶子节点
```

### 原因3: 【可能】多特征时的序列创建问题
```
特征矩阵拆平示意:
[Close_t, Close_t-1, ..., Close_t-29, High_t, High_t-1, ..., High_t-29]
↑ 共60个特征

问题:
- 60维特征对决策树来说容易过度拟合
- 特别是在测试集上，特征组合可能没有见过
- 导致大量测试样本流到相同的叶子节点
```

---

## ✅ 解决方案

### 方案1: 【推荐】回到单特征 (Close Only)
```python
# 改为只用Close价格，不用多特征
# 这样决策树更容易学习
- 0288.HK: 改用 Close （而不是 Close 单独已经很好了）
- 0005.HK: 改用 Close （而不是 Close+High）

预期效果: 避免特征维度过高导致的过度拟合
```

### 方案2: 改进决策树超参数
```python
# 当前
DecisionTreeRegressor(max_depth=10, min_samples_split=5)

# 改进选项1 - 增加正则化强度
DecisionTreeRegressor(
    max_depth=5,              # 降低深度，防止过度拟合
    min_samples_split=20,     # 提高最小分割样本数
    min_samples_leaf=10,      # 设置最小叶子样本数
    random_state=42
)

# 改进选项2 - 中等正则化
DecisionTreeRegressor(
    max_depth=8,
    min_samples_split=10,
    min_samples_leaf=5,
    random_state=42
)
```

### 方案3: 放弃决策树模型（对这两个股票）
```
由于决策树在这两个股票上表现极差，
而线性回归和LSTM已经表现很好，
可以考虑:
- 只用线性回归 + LSTM
- 对于某些股票，不强制用三个模型
```

### 方案4: 【完整方案】同时做多个改进
```
1. 禁用多特征对决策树的影响（只用Close）
2. 调整决策树超参数
3. 添加验证逻辑 - 如果决策树预测只有<10个唯一值，发出警告
```

## 💡 深度诊断

让我检查一个可能的问题 - 0005.HK的特征信息：

```

- 这是树无法适应测试数据的表现
```

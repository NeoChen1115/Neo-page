The topic of project is about prediction analytics using public or your own collected data. You have
to use predictive models that forecast future trends in a chosen domain (e.g., stock market,
consumer behavior, social media, etc.) using historical data. Your group has to compare different
forecasting models to determine the most effective approach for their specific use case.
We require students to use Python through out this project, you are free to use any 3rd party Python
packages.
Project Scope:
In the project, you should go through the following steps to approach all the deliverables described
in the next section.
1. Domain Selection and Dataset Collection:
o Choose a domain for trend prediction (e.g., stock market, fashion, energy
consumption).
.
o Justify the choice based on data availability and collect relevant datasets from public
sources such as Kaggle, UCI Machine Learning Repository, Hong Kong government
open data platform, or other resources.
2. Discovery of Data Characteristics and Preprocessing
o Perform a preliminary understanding of the data characteristics with descriptive
statistics & visualization and identify key trends.
o Clean and preprocess the data, handling missing values, outliers, and data
normalization if necessary.
3. Model Development:
o Select and implement multiple forecasting models (e.g., linear regression, ARIMA,
SARIMA, Meta Prophet, Decision Tree Regression, LSTM, and etc).
o Train and validate each model using historical data.
4. Model Comparison and Evaluation:
o Evaluate model performance with validation/testing dataset
o Choose an error metrics (e.g. Mean Squared Error (MSE), Mean Absolute Percentage
Error (MAPE) or etc) to quantify the performance of your model
o Compare and discuss the results of different models to identify the most effective
one.
5. Prediction and Visualization:
o Use the best-performing model to predict future trends and visualize the results.
o Provide insights and recommendations based on the predictions.
6. Report and Presentation:
o Prepare a comprehensive report detailing the methodology, analysis, and findings.
o Present the project outcomes, highlighting key insights, challenges, and the rationale
behind model selection.
.

1. Report Package (10/15 marks):
• Data and Code (2/10 marks):

report and presentation.
o Requirements: Include a README file with instructions on how to run your code.
NO marks will be awarded if your code cannot be executed properly.

o Provide a clear, well-organized report covering all project aspects. The report
should include sections/contents similar to the following:
▪ Cover page: include a title, your member list with student IDs and names
▪ Introduction: Overview of the project objectives and significance.
▪ Data Source and Transformation: Describe your data source, the critical

clearly the models attempted and the method for evaluation.

recommendation and challenges (if any).
o The report is expected to include proper figures and explanations and should NOT
exceed 25 pages.
o Evaluation Criteria: The report will be evaluated based on the depth and clarity of
insights gained from the data analysis process, the data collection challenges (if
applicable), the thoroughness of the analysis steps and trials, validation of the model,

selection, and the overall coherence of the report.
